# 心语实施计划文档

## 1. 交付策略

MVP 必须按严格依赖顺序构建：

1. 先冻结规格；
2. 再搭建后端与数据库基础；
3. 再录入题库与评分逻辑；
4. 再实现学生端主流程；
5. 再实现 AI 与风险闭环；
6. 再实现管理后台；
7. 最后完成整条答辩演示链路测试。

在问卷评分、风险规则和工单状态稳定之前，不应优先做大量 UI 装饰工作。

## 2. 第一阶段：仓库初始化

### 1.1 创建项目目录

- 创建根目录结构：
  - `backend/`
  - `admin/`
  - `miniprogram/`
  - `appendices/question_bank/`
  - `tests/`
- 完成标准：
  - 所有目录已存在；
  - 根目录规范文档已落盘。

### 1.2 初始化 Python 项目

- 创建 `backend/pyproject.toml`
- 按 `TECH_STACK.md` 锁定运行时依赖
- 按 `TECH_STACK.md` 锁定开发依赖
- 完成标准：
  - `uv` 或 `pip` 安装成功；
  - 虚拟环境可运行 FastAPI 与 Streamlit。

### 1.3 配置代码质量工具

- 配置 `ruff`
- 配置 `black`
- 配置 `pytest`
- 完成标准：
  - lint、format、test 命令能正常执行

### 1.4 初始化小程序外壳

- 创建原生微信小程序项目
- 补齐基础文件：
  - `app.js`
  - `app.json`
  - `app.wxss`
- 创建页面目录与共享组件目录
- 完成标准：
  - 微信开发者工具可正常打开项目；
  - 空壳导航能够正常跳转。

## 5. 第四阶段：题库附录与种子数据

### 4.1 定义题库种子文件结构

- 为题库文件定义统一 JSON 结构
- 必填字段：
  - `question_id`
  - `question_text`
  - `question_type`
  - `options`
  - `score_mapping`
  - `reverse_scored`
  - `hard_trigger_rule`
- 完成标准：
  - 有一个校验脚本可以验证所有题库文件格式。

### 4.2 写入项目自定义问卷种子

- 创建：
  - `appendices/question_bank/screen_questions.json`
  - `appendices/question_bank/sleep_questions.json`
  - `appendices/question_bank/upi_questions.json`
- 完成标准：
  - 三份自定义题库与冻结需求完全一致。

### 4.3 写入 SDS 与 SAS 种子

- 创建：
  - `appendices/question_bank/sds_questions.json`
  - `appendices/question_bank/sas_questions.json`
- 必须包含反向计分与硬触发定义
- 完成标准：
  - SDS / SAS 评分可完全依赖题库元数据计算。

### 4.4 实现种子导入脚本

- 编写统一导入脚本，负责插入或刷新：
  - 问卷模板
  - 题库条目
- 完成标准：
  - 空数据库可一次性完成问卷初始化。

## 6. 第五阶段：评分与报告引擎

### 5.1 实现各问卷评分服务

- 实现以下评分器：
  - 快速筛查
  - SDS
  - SAS
  - 睡眠问卷
  - UPI
- 完成标准：
  - 单元测试覆盖所有阈值边界。

### 5.2 实现硬触发判断逻辑

- 必须覆盖：
  - `SCREEN_15 >= 4`
  - SDS 第 15 题 `>= 4`
  - SAS 第 13 题 `>= 4`
  - `UPI_01=yes`
  - `UPI_02=yes`
- 完成标准：
  - 命中硬触发后，正常评分流程被提升为高风险。

### 5.3 实现报告生成逻辑

- 生成：
  - 单量表结果
  - 完整综合画像报告
- 强制执行解锁规则：
  - 快速筛查 + SDS + SAS + 睡眠问卷全部完成
- 完成标准：
  - 只有完成 70 道必做题时，完整报告才可见。

## 10. 第九阶段：AI 集成与风险引擎

### 9.1 实现 DeepSeek 客户端

- 使用 `httpx`
- 接口地址：`https://api.deepseek.com`
- 模型：`deepseek-v4-flash`，请求中显式关闭思维链模式
- 强制要求 JSON 输出
- 完成标准：
  - 常规调用时后端能拿到可解析 JSON。

### 9.2 实现本地模拟分析路径

- 添加本地 `mock_response.json`
- 在以下场景使用本地模拟分析：
  - 超时
  - 非法 JSON
  - 非 200 响应
- 完成标准：
  - AI 服务异常时，树洞主流程不会中断。

### 9.3 实现风险聚合器

- 聚合以下输入：
  - AI 文本分析结果
  - 问卷分数
  - 硬触发规则
  - 历史风险状态
- 完成标准：
  - 低风险 / 需关注 / 高风险判定与冻结规格一致。

### 9.4 实现发布决策逻辑

- 低风险 -> 发布
- 需关注 -> 发布并写重点关注列表
- 高风险 -> 拦截并创建工单
- 完成标准：
  - 高风险内容绝不出现在公开广场。

## 16. 完成定义

只有当以下条件全部满足时，实施阶段才算完成：

- 六份规范文档内部没有互相冲突；
- 所有问卷种子文件已存在；
- 70 题解锁规则已实现；
- 可选 UPI 不阻塞完整报告；
- 树洞低风险 / 需关注 / 高风险处理逻辑完全符合规格；
- 管理员确认流程可写入干预与审计日志；
- 本地模拟 AI 分析路径已被测试；
- 至少完成一次无需手工改库的完整答辩演练。
