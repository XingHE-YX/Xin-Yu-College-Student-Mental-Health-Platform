# 心语 PRD

## 1. 文档控制信息

- 产品名称：心语大学生心理健康互助平台
- 文档类型：产品需求文档
- 版本号：1.0
- 文档状态：MVP 实施前冻结版
- 适用范围：学生端微信小程序 + Python API + Streamlit 管理后台
- 最后更新时间：2026-04-04

## 2. 产品愿景

心语是一款面向高校本科生的校园心理健康支持平台。MVP 版本不是诊断系统，也不能以任何形式对外宣称自己具备诊断能力。它的目标是提供：

- 低门槛的心理自测能力；
- 保护隐私的匿名情绪表达空间；
- AI 辅助的早期风险筛查；
- 高风险信号的人审闭环干预流程。

本产品在答辩场景中要证明的核心只有一件事：从学生输入，到 AI 筛查，到人工复核，到模拟干预记录，整个闭环流程是可以跑通、可以解释、可以自圆其说的。

## 3. MVP 产品目标

MVP 必须完整演示如下闭环：

1. 学生完成量表，和/或提交树洞内容。
2. 系统对结构化量表进行评分，对非结构化文本进行分析。
3. 系统将结果划分为低风险、需关注、高风险。
4. 高风险内容被阻止公开发布，并进入后台人工复核队列。
5. 管理员完成复核，并记录一条模拟干预动作。

## 4. 目标用户

### 4.1 主要用户

- 单一校园场景下的中国籍本科生；
- 正处于心理压力状态、需要自我了解、情绪宣泄或低门槛支持的学生。

### 4.2 次级用户

- 模拟学校端管理员；
- 在 MVP 中，这个角色统一承载辅导员 / 心理老师 / 平台管理员的功能。

## 5. 产品原则

1. 安全性优先于传播性。
2. 人工复核优先于全自动干预。
3. 前台匿名，后台受控可追溯。
4. 产品边界清晰，避免医疗化过度承诺。
5. 演示稳定性优先于功能铺得过宽。

## 6. 成功标准

只有当以下条件全部满足时，MVP 才算成功：

- 必做测评链路总题数固定为 70 题：
  - 快速筛查：15 题
  - SDS：20 题
  - SAS：20 题
  - 睡眠问卷：15 题
- 可选 UPI 不影响完整报告解锁。
- 一条低风险树洞内容可以完成分析并自动公开发布。
- 一条高风险树洞内容会被拦截，并生成最高优先级待复核工单。
- 管理员可以确认或驳回一条工单，并填写干预记录。
- 任意敏感后台详情查看动作都会写入审计日志。
- 当 DeepSeek 调用失败时，系统可以回退到本地模拟分析，演示流程仍可完整跑通。

## 7. 非目标

MVP 明确不做以下内容：

- 真实精神疾病诊断；
- 真实医院转诊对接；
- 真实短信、电话等外部通知；
- 学生之间自由私信；
- 学生之间自由文本评论；
- 在线心理咨询预约；
- 真实校内信息系统打通；
- 多学校大规模部署；
- 自行训练深度学习模型。

## 9. 功能范围

### 9.1 学生端小程序

- 微信登录与一键手机号授权
- 危机干预授权同意
- 快速筛查问卷
- SDS 问卷
- SAS 问卷
- 睡眠问卷
- 可选 UPI 问卷
- 测评历史记录
- 各量表单独结果页
- 完整报告解锁逻辑
- 树洞发帖
- 树洞广场
- 树洞前端删除
- 安全提示弹窗与求助引导
- 静态帮助资源页

### 9.2 管理后台

- 管理员登录
- 数据总览看板
- 预警工单队列
- 预警详情页
- 确认 / 驳回复核操作
- 模拟干预记录
- 树洞内容管理
- 用户目录与脱敏信息查看
- 趋势分析图表
- 审计日志页

### 9.3 AI 与规则引擎

- 使用 DeepSeek 对树洞文本进行分析
- 强制结构化 JSON 输出
- 将文本风险、量表分数、硬触发规则、历史记录合并判定
- 外部接口失败时回退到本地模拟结果

### 9.4 数据治理

- 同意书记录
- 树洞软删除
- 后台列表页敏感数据脱敏
- 敏感访问审计日志
- 不强制收集真实姓名

## 10. 测评体系规范

### 10.1 必做与可选问卷

| 问卷 | 是否必做 | 题量 | 评分方式 |
| :--- | :--- | :--- | :--- |
| 快速筛查 | 是 | 15 | 每题 1-5 分，总分 15-75 |
| SDS | 是 | 20 | Zung 标准分算法 |
| SAS | 是 | 20 | Zung 标准分算法 |
| 睡眠问卷 | 是 | 15 | 每题 0-3 分，总分 0-45 |
| UPI | 否 | 4 | 是/否辅助判断 |

### 10.2 报告解锁逻辑

- 完整报告解锁条件：完成快速筛查、SDS、SAS、睡眠问卷。
- 必做总题数固定为 70 题。
- UPI 为可选问卷，不参与完整报告解锁。

### 10.3 固定评分规则

#### SDS

- 4 级评分；
- 反向计分题号：2、5、6、11、12、14、16、17、18、20；
- 反向计分公式：`5 - selected_score`；
- 粗分 `X = 各题得分之和`；
- 标准分 `Y = int(1.25 * X)`；
- 阈值：
  - 低风险：`< 53`
  - 需关注：`53-62`
  - 高风险：`>= 63`

#### SAS

- 4 级评分；
- 反向计分题号：5、9、13、17、19；
- 反向计分公式：`5 - selected_score`；
- 粗分 `X = 各题得分之和`；
- 标准分 `Y = int(1.25 * X)`；
- 阈值：
  - 低风险：`< 50`
  - 需关注：`50-59`
  - 高风险：`>= 60`

#### 睡眠问卷

- 每题 0-3 分；
- 总分范围：`0-45`；
- 阈值：
  - 低风险：`0-7`
  - 需关注：`8-14`
  - 高风险：`>= 15`

#### 快速筛查

- 每题 1-5 分；
- 总分范围：`15-75`；
- 阈值：
  - 低风险：`< 45`
  - 需关注：`>= 45`
- 快速筛查本身不直接生成带有医疗意味的公开结论。

#### UPI

- 可选；
- 不参与完整报告解锁；
- 不参与综合总报告总分；
- 仅作为辅助参考与风险升级信号。

## 11. 硬触发规则

以下规则一旦命中，会跳过一般聚合逻辑，直接提升为高风险：

| 规则 ID | 来源 | 条件 | 结果 |
| :--- | :--- | :--- | :--- |
| HT-01 | 快速筛查 | `SCREEN_15 >= 4` | 高风险 |
| HT-02 | SDS | 官方第 15 题回答 `>= 4` | 高风险 |
| HT-03 | SAS | 官方第 13 题回答 `>= 4` | 高风险 |
| HT-04 | UPI | `UPI_01 = yes` | 高风险 |
| HT-05 | UPI | `UPI_02 = yes` | 高风险 |
| HT-06 | 树洞 AI | AI 返回 `risk_level = high` | 高风险 |

## 12. 风险分级规则

### 12.1 数据完整时

- 低风险：
  - 没有命中硬触发；
  - 没有高风险 AI 结果；
  - SDS 与 SAS 均低于阳性阈值；
  - 睡眠得分低于需关注阈值；
  - 历史记录没有把当前案例拉升。
- 需关注：
  - SDS 或 SAS 触发阳性阈值；
  - 或睡眠分数为 `8-14`；
  - 或 AI 识别为持续消极情绪，但没有自杀 / 自残意图；
  - 或用户有历史高风险记录，而本次内容中性。
- 高风险：
  - 任意硬触发命中；
  - 或 AI 识别到自杀 / 自残意图；
  - 或 SDS `>= 63`；
  - 或 SAS `>= 60`。

### 12.2 数据不完整时

- 只有树洞、没有量表：
  - AI 高风险 -> 高风险
  - AI 消极但不涉自杀 / 自残 -> 需关注
- 只有量表、没有树洞：
  - SDS / SAS 任一阳性 -> 需关注
  - SDS `>= 63` 或 SAS `>= 60` -> 高风险
- 历史高风险，本次内容中性：
  - 标记为需关注，并在后台加入复查提示
- 对于需关注级别：
  - 不发送外部通知
  - 不弹出强烈安全警报
  - 仅写入后台重点关注列表

## 13. 树洞规则

### 13.1 发布规则

- 低风险内容：分析后自动发布
- 需关注内容：允许发布，但后台进入重点关注列表
- 高风险内容：不允许公开发布

### 13.2 互动规则

- 允许：
  - 预设的非文本支持反馈，如抱抱 / 点亮 / 陪伴
- 不允许：
  - 自由文本评论
  - 私信
  - 学生间交换联系方式

### 13.3 删除规则

- 学生可以在前端删除自己发过的内容
- 后端只执行软删除
- 管理员在受限权限和审计条件下仍可查看软删除内容

## 14. 隐私与授权规则

- 其他学生只能看到匿名昵称和头像
- 后台保留受控手机号映射
- 不强制采集真实姓名
- 采集学院与班级作为结构化字段
- 首次使用时必须展示危机干预授权
- 如果用户拒绝授权：
  - 允许做单次测评；
  - 禁止发布树洞；
  - 不允许进入与干预相关的功能链路。

## 15. 内容治理规则

- AI 同时承担合规过滤职责，用于识别涉黄、辱骂、隐私暴露、违法等内容；
- 涉及第三方身份暴露的内容必须被拦截或隐藏；
- 管理员可以执行隐藏、维持隐藏、恢复发布等操作；
- 任何敏感详情展开动作都必须先确认，再记录审计日志。

## 16. 后台处置流程规则

- 高风险案例进入 `pending_review`
- 管理员可执行：
  - 确认高风险；
  - 标记为误报
- 管理员确认后，系统只写“模拟通知日志”，不进行真实外发通知；
- 案例状态流转：
  - `pending_review`
  - `confirmed_pending_intervention` 或 `dismissed_false_positive`
  - `closed`

## 17. 演示模式规则

- 学生端仅在环境变量显式开启时显示演示登录入口；
- AI 外部调用失败时，使用本地 `mock_response.json` 作为回退；
- 后台预置低风险 / 需关注 / 高风险演示数据。

## 18. 验收标准

只有当以下验收条件全部通过时，MVP 才算准备完成：

- AC-01：学生可以完成 70 道必做题并解锁完整报告。
- AC-02：可选 UPI 被跳过时，不影响报告解锁。
- AC-03：低风险树洞内容可以进入公开广场。
- AC-04：高风险树洞内容会被拦截，不能出现在广场。
- AC-05：管理员可以对高风险案例进行复核并写入干预记录。
- AC-06：后台敏感详情查看会写入审计日志。
- AC-07：AI 服务异常时，模拟回退路径可用，不影响答辩演示。
- AC-08：任何界面文案都不能宣称系统具备诊断或治疗确定性。
