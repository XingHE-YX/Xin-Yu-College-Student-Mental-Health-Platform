# 心语应用流程文档

## 1. 全局规则

- 学生端应用形态为微信小程序。
- 管理端应用形态为 Streamlit 网页后台。
- 必做测评解锁链路固定为 70 题。
- 可选 UPI 不阻塞完整报告解锁。
- 高风险树洞内容绝不进入公开广场。
- 后台任何敏感详情查看都必须写入审计日志。

## 2. 学生端小程序页面流程

### S01. 启动入口页

- 触发条件：用户打开小程序。
- 进入后检查：
  - 是否存在本地有效会话；
  - 是否已完成授权同意；
  - 是否开启演示模式。
- 成功路径：
  - 会话有效 -> 进入 `S04 首页`
  - 无有效会话 -> 进入 `S02 登录页`
- 错误状态：
  - 本地存储读取失败 -> 清空本地令牌并跳转 `S02 登录页`

### S02. 登录页

- 页面目标：创建或刷新学生会话。
- 触发条件：用户没有有效登录状态。
- 主要动作：
  - 点击 `Login with WeChat`
  - 获取 `wx.login` code
  - 获取手机号授权
  - 调用后端登录接口
- 成功状态：
  - 后端返回学生信息与会话令牌
  - 若缺少授权记录 -> 跳转 `S03 授权页`
  - 否则 -> 跳转 `S04 首页`
- 错误状态：
  - 用户拒绝手机号授权 -> 展示阻断提示并停留当前页
  - 微信登录失败 -> 显示重试按钮
  - 后端登录失败 -> 显示重试按钮

### S03. 授权页

- 页面目标：记录危机干预授权同意结果。
- 触发条件：首次登录且尚未保存授权记录。
- 主要动作：
  - 阅读隐私说明
  - 阅读危机干预授权说明
  - 选择 `Agree` 或 `Decline`
- 成功状态：
  - 同意 -> 保存授权结果并跳转 `S04 首页`
  - 拒绝 -> 保存拒绝记录并跳转 `S04 首页`，同时禁用树洞功能
- 错误状态：
  - 授权结果保存失败 -> 停留当前页并允许重试

### S04. 首页

- 页面目标：学生端主导航入口。
- 进入条件：学生会话有效。
- 页面模块：
  - 快速筛查
  - 深度测评中心
  - 树洞
  - 我的报告
  - 帮助资源
- 动态状态：
  - 若用户拒绝授权，树洞入口为禁用态
  - 若完整报告未解锁，报告卡片显示未完成进度
- 导航路径：
  - 快速筛查 -> `S05 快速筛查`
  - SDS -> `S06 SDS`
  - SAS -> `S07 SAS`
  - 睡眠问卷 -> `S08 睡眠问卷`
  - 可选 UPI -> `S09 UPI`
  - 我的报告 -> `S10 报告页`
  - 树洞广场 -> `S11 树洞广场`
  - 帮助资源 -> `S15 帮助页`

### S05. 快速筛查页

- 页面目标：完成 15 题快速心理筛查。
- 触发条件：用户从首页点击快速筛查。
- 执行步骤：
  1. 加载问卷元数据
  2. 渲染 15 道题目
  3. 收集单选答案
  4. 提交作答结果
  5. 展示快速筛查结果摘要
- 成功状态：
  - 本次作答成功保存
  - 展示简要结果说明
  - 更新 70 题必做进度
- 错误状态：
  - 问卷加载失败 -> 显示重试
  - 未完成全部作答 -> 阻止提交并高亮未答题
  - 保存失败 -> 在不丢失本地答案的前提下允许重试

### S06. SDS 页

- 页面目标：完成必做抑郁量表。
- 触发条件：用户从首页或快速筛查结果页进入 SDS。
- 执行步骤：
  1. 加载 20 道正式题目
  2. 完成 4 级评分
  3. 调用评分接口提交
  4. 展示 SDS 结果页
- 成功状态：
  - 计算并返回标准分
  - 展示分级结果
  - 更新进度
- 错误状态：
  - 存在未答题目 -> 禁止提交
  - 评分接口失败 -> 允许重试

### S07. SAS 页

- 页面目标：完成必做焦虑量表。
- 触发条件：用户从首页或承接流程进入 SAS。
- 执行步骤与 `S06 SDS` 结构一致。
- 成功状态：
  - 返回标准分
  - 展示风险分级
  - 更新进度
- 错误状态：
  - 与 `S06 SDS` 相同

### S08. 睡眠问卷页

- 页面目标：完成必做睡眠与作息问卷。
- 触发条件：用户从首页或承接流程进入睡眠问卷。
- 执行步骤：
  1. 加载 15 道题目
  2. 完成 0-3 映射题目作答
  3. 提交问卷
  4. 展示睡眠结果页
- 成功状态：
  - 计算睡眠总分
  - 展示低风险 / 需关注 / 高风险结果
  - 更新进度
- 错误状态：
  - 与其他必做问卷一致

### S09. UPI 页

- 页面目标：完成可选辅助筛查。
- 触发条件：用户主动点击 UPI 入口。
- 执行步骤：
  1. 加载 4 道是/否题
  2. 提交答案
  3. 展示“仅作辅助参考”的说明
- 成功状态：
  - 结果被保存
  - 后台可将其作为辅助风险参考
- 错误状态：
  - 与其他问卷一致
- 特别说明：
  - 跳过此页不能影响完整报告解锁

### S10. 报告页

- 页面目标：展示量表结果与完整报告解锁状态。
- 触发条件：用户打开“我的报告”。
- 页面状态：
  - 锁定状态：必做 70 题尚未完成
  - 部分完成状态：已有部分量表结果
  - 完整解锁状态：必做问卷全部完成
- 导航逻辑：
  - 锁定状态 -> 可跳转至未完成问卷
  - 已解锁状态 -> 可进入 `S10A 完整报告详情`

### S10A. 完整报告详情页

- 页面目标：展示完整综合心理画像报告。
- 进入条件：快速筛查、SDS、SAS、睡眠问卷均已完成。
- 页面内容：
  - 综合结果标题
  - 各量表摘要
  - 趋势占位区
  - 调节建议
  - 明确写明“本结果不构成诊断”
- 错误状态：
  - 若解锁条件不满足，返回 `S10 报告页`

### S11. 树洞广场页

- 页面目标：显示公开匿名树洞内容。
- 触发条件：用户从首页打开树洞模块。
- 进入规则：
  - 若用户拒绝危机授权，则展示禁用状态，不允许发帖
  - 否则加载已发布内容
- 主要动作：
  - 浏览广场
  - 打开帖子详情
  - 点击预设反应
  - 点击发帖按钮
- 成功状态：
  - 低风险 / 需关注内容可正常显示
  - 预设反应计数更新
- 错误状态：
  - 广场加载失败 -> 显示重试
  - 反应提交失败 -> 回滚前端状态

### S12. 树洞发布页

- 页面目标：提交匿名情绪表达内容。
- 触发条件：用户从 `S11 树洞广场页` 点击发帖。
- 执行步骤：
  1. 输入文本内容
  2. 点击提交
  3. 后端保存原始文本
  4. 后端执行内容合规与风险分析
  5. 返回最终发帖处理结果
- 成功状态：
  - 低风险 -> 跳转 `S13 发布成功`
  - 需关注 -> 跳转 `S13 发布成功`
  - 高风险 -> 跳转 `S14 安全拦截`
- 错误状态：
  - 内容为空 -> 禁止提交
  - 内容违规 -> 展示合规拦截提示
  - AI 调用失败 -> 走模拟回退路径
  - 后端保存失败 -> 显示重试

### S13. 发布成功页

- 页面目标：处理非高风险树洞发布结果。
- 进入条件：`S12 树洞发布页` 返回低风险或需关注。
- 页面反馈：
  - 低风险 -> 显示“发布成功”
  - 需关注 -> 显示“发布成功”，并可附带温和的自我调节建议
- 导航动作：
  - 返回广场
  - 查看我的帖子

### S14. 安全拦截页

- 页面目标：处理高风险树洞内容。
- 进入条件：`S12 树洞发布页` 被判定为高风险。
- 页面行为：
  - 不发布该内容
  - 展示严肃安全提醒
  - 展示校园求助电话
  - 提示联系可信赖的人
- 成功状态：
  - 后台成功创建高优先级工单
  - 学生端进入安全提示流程
- 错误状态：
  - 工单创建失败 -> 提示重试，同时保留本地内容直到保存成功

### S15. 帮助页

- 页面目标：提供静态求助资源。
- 触发条件：用户从首页或安全拦截页进入。
- 页面内容：
  - 校园求助电话
  - 呼吸调节建议
  - 校内心理中心信息
  - 紧急情况处理指引
- 错误状态：
  - 仅在静态资源加载失败时出现

### S16. 我的帖子详情页

- 页面目标：查看自己的帖子状态并允许删除。
- 触发条件：用户从广场或个人记录进入自己的帖子详情。
- 主要动作：
  - 查看当前状态
  - 删除帖子
- 成功状态：
  - 删除动作成功后，帖子从学生端列表中消失
- 错误状态：
  - 删除失败 -> 保持当前可见并提示重试

## 3. 管理后台页面流程

### A01. 管理员登录页

- 触发条件：管理员在没有有效令牌的情况下打开后台。
- 执行步骤：
  1. 输入用户名
  2. 输入密码
  3. 提交认证
  4. 获取 JWT 会话
- 成功状态：
  - 跳转 `A02 总览页`
- 错误状态：
  - 用户名或密码错误 -> 行内提示
  - 账户被禁用 -> 显示拒绝访问状态

### A02. 总览页

- 页面目标：提供后台概览。
- 页面内容：
  - 待复核工单数
  - 高风险工单数
  - 重点关注人数
  - 已发布帖子数
  - 基础统计卡片
- 导航逻辑：
  - 待复核 -> `A03 预警队列`
  - 树洞帖子 -> `A05 帖子管理`
  - 用户 -> `A06 用户目录`
  - 审计 -> `A07 审计日志`

### A03. 预警队列页

- 页面目标：按优先级处理待复核案例。
- 默认排序：
  - 最高优先级在前
  - 同优先级按时间倒序
- 可筛选状态：
  - 待复核
  - 已确认
  - 已忽略
  - 已结案
- 成功状态：
  - 点击任一工单可进入 `A04 预警详情`
- 错误状态：
  - 列表加载失败 -> 显示重试

### A04. 预警详情页

- 页面目标：对单条案例进行人工复核。
- 进入条件：
  - 管理员从队列或重点关注列表选择一条案例
- 强制审计行为：
  - 打开敏感详情必须写入审计日志
- 页面内容：
  - 列表上下文默认展示脱敏身份
  - 完整原文需二次确认后才显示
  - AI 风险等级
  - AI 触发短语
  - 历史测评信息
  - 历史风险标记
  - 处置动作区
- 可执行动作：
  - 确认高风险
  - 标记误报
  - 添加干预记录
  - 结案
- 成功状态：
  - 确认 -> 写入模拟通知日志并推进工单状态
  - 误报 -> 保存复核意见并更新状态
  - 结案 -> 进入最终关闭状态
- 错误状态：
  - 权限不足
  - 该工单已被其他管理员处理
  - 写入失败

### A05. 帖子管理页

- 页面目标：管理树洞内容。
- 页面内容：
  - 已发布帖子
  - 被拦截帖子
  - 已删除帖子
  - 脱敏与展开查看流程
- 可执行动作：
  - 隐藏帖子
  - 保持隐藏
  - 恢复发布
  - 查看软删除内容
- 审计要求：
  - 展开敏感详情和导出操作必须写审计日志

### A06. 用户目录页

- 页面目标：以受控方式查看学生风险档案。
- 默认展示：
  - 匿名 ID
  - 脱敏手机号
  - 学院 / 班级
  - 最新风险状态
- 详情查看：
  - 必须显式触发“查看详情”
  - 需要二次确认
  - 展开后可看到完整手机号
  - 展开动作必须写审计日志

### A07. 审计日志页

- 页面目标：展示敏感操作全链路记录。
- 可筛选条件：
  - 操作人
  - 操作类型
  - 时间范围
  - 目标对象类型
- 成功状态：
  - 列表可展示时间、操作者、目标对象和摘要信息

## 4. 关键端到端流程

### F01. 必做测评完成流程

1. 用户打开小程序。
2. 用户完成登录并处理授权。
3. 用户完成快速筛查。
4. 用户完成 SDS。
5. 用户完成 SAS。
6. 用户完成睡眠问卷。
7. 系统标记 `70/70 required questions complete`。
8. 完整报告入口可用。

### F02. 低风险树洞发布流程

1. 用户打开树洞发布页。
2. 用户提交内容。
3. 后端保存原始文本。
4. DeepSeek 或模拟路径返回低风险。
5. 后端脱敏并标记为已发布。
6. 用户看到发布成功页。
7. 广场出现新帖子。

### F03. 高风险树洞拦截流程

1. 用户提交包含高风险信号的内容。
2. 后端保存原始文本。
3. AI 或硬规则判定为高风险。
4. 后端阻止发布。
5. 后端生成最高优先级待复核工单。
6. 学生端展示安全提醒与求助电话。
7. 后台队列收到新工单。

### F04. 管理员复核与模拟干预流程

1. 管理员登录后台。
2. 打开预警队列。
3. 选择一条高风险工单。
4. 系统为详情展开动作写入审计日志。
5. 管理员查看 AI 原因与原始内容。
6. 管理员确认高风险。
7. 系统写入模拟通知日志。
8. 管理员填写干预记录。
9. 工单进入已确认或已结案状态。

## 5. 全局错误处理模式

- 学生端所有提交失败都必须保留本地表单状态；
- 当 AI 接口失败且启用了模拟回退时，不得中断学生主流程；
- 管理后台出现并发写冲突时，必须展示最新状态，不能静默覆盖；
- 任意身份认证失败都必须跳回对应登录页。
