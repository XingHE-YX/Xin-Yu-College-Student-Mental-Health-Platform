# 心语前端指南（旧版 UI/UX 设计归档）

> 文档状态：本文件保留为上一版 / 旧版 UI/UX 设计基线，用于追溯 MVP 初始视觉系统、旧版设计令牌和旧版组件约束。
>
> 当前拟执行的 UI/UX 重绘方案不直接覆盖本文件正文，另见 `New_FRONTEND_GUIDELINES.md` 中的“当前 UI/UX 重绘实施方案（待确认）”。在新版方案确认并落地前，本文件仍可作为旧版维护与回归比对依据。

## 1. 设计目标

学生端小程序的整体气质必须是平静、可信、低压的；管理后台则必须克制、信息密度高、强调可审计。整体视觉不能落入“心理健康 = 过度卡通化”的套路，但也不能显得冷硬和疏离。

## 2. 适用平台

- 学生端：微信小程序
- 管理端：Streamlit 网页后台

## 3. 设计原则

1. 平静优先于花哨。
2. 风险提示的视觉层级必须高于普通通知。
3. 手机端核心操作必须一眼可见。
4. 高风险状态绝不能用游戏化、趣味化方式表达。
5. 树洞社区应当温和，但不能显得杂乱。

## 5. 字体系统

### 5.1 学生端

- 中文主字体栈：
  - `PingFang SC`
  - `Hiragino Sans GB`
  - `Microsoft YaHei`
  - `sans-serif`
- 数字强调字体：
  - `DIN Alternate`
  - 回退到系统 sans-serif

### 5.2 管理端

- 正文字体：
  - `Inter`
  - `PingFang SC`
  - `Microsoft YaHei`
  - `sans-serif`
- 数字与指标字体：
  - `DIN Alternate`
  - `SF Mono`
  - `monospace`

## 8. 布局规则

### 8.1 学生端

- 页面左右安全边距：`16px`
- 手机端内容最大宽度：`100%`
- 区块间距：`24px`
- 卡片内边距：`16px`
- 最小点击区域：`44px x 44px`

### 8.2 管理端

- 内容最大宽度：`1280px`
- 标准布局间距：`24px`
- KPI 卡片间距：`16px`
- 表格行最小高度：`44px`

## 9. 响应式断点

### 9.1 学生端

| 断点 | 范围 | 规则 |
| :--- | :--- | :--- |
| `xs-phone` | `<= 360px` | 仅单列布局，卡片收紧，指标不并排 |
| `phone` | `361px - 430px` | 默认设计目标尺寸 |
| `large-phone` | `431px - 767px` | 可允许双列摘要卡片 |
| `tablet` | `>= 768px` | 内容居中，报告页可放宽布局 |

### 9.2 管理端

| 断点 | 范围 | 规则 |
| :--- | :--- | :--- |
| `desktop-lg` | `>= 1280px` | 4 列 KPI 卡片，宽表格布局 |
| `desktop` | `1024px - 1279px` | 3 列 KPI 卡片 |
| `tablet` | `768px - 1023px` | 筛选条件上下堆叠，2 列 KPI |
| `mobile-web` | `< 768px` | 不推荐后台移动使用，但必须保证可读 |

## 10. 组件策略

### 10.1 学生端

- 不引入第三方组件库；
- 使用微信小程序原生组件 + 项目内业务组件。

必须实现的业务组件：

- `app-shell`
- `hero-card`
- `primary-button`
- `secondary-button`
- `danger-button`
- `question-card`
- `progress-bar`
- `result-badge`
- `safety-banner`
- `treehole-post-card`
- `reaction-chip`
- `confirm-modal`

### 10.2 管理端

- 仅使用 Streamlit 原生组件；
- 如需统一视觉，只能通过主题配置与局部 CSS 注入实现。

## 11. 核心组件规范

### 11.1 按钮

| 类型 | 背景色 | 文字色 | 圆角 | 高度 |
| :--- | :--- | :--- | :--- | :--- |
| 主按钮 | `#2F8F83` | `#FFFFFF` | `14px` | `48px` |
| 次按钮 | `#EEF7F5` | `#1E645C` | `14px` | `48px` |
| 危险按钮 | `#D84C4C` | `#FFFFFF` | `14px` | `48px` |
| Ghost 按钮 | `transparent` | `#2F8F83` | `14px` | `44px` |

禁用态：

- 背景：`#E6ECEA`
- 文字：`#98A4A1`
- 不使用阴影

### 11.2 输入框

- 高度：`48px`
- 背景：`#FFFFFF`
- 边框：`1px solid #D7E1DE`
- 聚焦边框：`#2F8F83`
- 错误边框：`#D84C4C`

### 11.3 问卷卡片

- 背景：`#FFFFFF`
- 圆角：`20px`
- 内边距：`20px`
- 题号角标使用 `brand-100`
- 每个选项行高度不得低于 `44px`

### 11.4 结果标签

| 状态 | 背景色 | 文字色 |
| :--- | :--- | :--- |
| 低风险 | `#E9F6ED` | `#2F7D49` |
| 需关注 | `#FFF3DF` | `#A86A14` |
| 高风险 | `#FBE7E7` | `#A92A2A` |

### 11.5 树洞帖子卡片

- 背景：`#FFFFFF`
- 圆角：`18px`
- 内边距：`16px`
- 头部包含头像、匿名昵称、相对时间
- 广场视图正文最多展示 6 行
- 互动区采用描边 Chip 形式

### 11.6 安全提示弹窗

- 背景：`#FFFFFF`
- 顶部强调边：`4px solid #D84C4C`
- 标题颜色：`#A92A2A`
- 主操作按钮必须使用危险色
- 必须展示热线电话与冷静处理提示
- 禁止使用玩偶化、卡通化插画

## 12. 动效规则

- 标准转场：`160ms ease-out`
- 弹窗出现：`240ms ease-out`
- 所有安全与风险相关界面禁止使用弹跳动效
- 进度条可做动画，但总时长不得超过 `320ms`

## 13. 图标规范

- 使用圆角化、低细节、克制风格图标
- 描边建议：`2px`
- 高风险图标必须使用语义红色
- 社区互动图标可以偏柔和，但不能使用霓虹色

## 14. 文案规范

- 禁止使用“你患有某种疾病”这一类诊断性措辞；
- 低风险文案可以温和支持；
- 需关注文案必须冷静、客观；
- 高风险文案必须直接、严肃、可行动；
- 树洞拦截文案不能带有羞辱或责备语气。

## 15. 可访问性要求

- 正文最小字号：`15px`
- 正文对比度目标：至少 `4.5:1`
- 交互控件最小尺寸：`44px`
- 风险状态不能只靠颜色区分，必须叠加图标或文字

## 16. 禁止项

MVP 前端禁止出现以下风格：

- 霓虹渐变主视觉；
- 泛紫色默认 SaaS 配色；
- 过度吉祥物化的心理主题形象；
- 聊天气泡堆满全页；
- 大量漂浮贴纸式装饰；
- 默认深色模式展示。
